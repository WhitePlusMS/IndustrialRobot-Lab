# 调研发现：PoE 解析运动学模型

## 2026-06-22 — 初始调研

### DH vs 场景几何

- **DH FK 与 GLB 场景 FK 偏差 1500-4400mm**，不是校准误差量级
- 根因：GLB 关节旋转轴为 Z/Y/Y/X/Y/Z（混合），DH 要求全 Z 轴
- 结论：纯 DH 替换不可行，必须以场景几何为准

### 场景关节轴实测

在零位转动每个关节 1°，从旋转矩阵差提取世界系下的旋转轴：

| 关节 | 世界旋转轴 | 末端位移(mm/°) |
|------|-----------|----------------|
| J1 | [0, 0.9999, 0.01] ≈ Y 轴 | 21.5 |
| J2 | [0, 0.01, -0.9999] ≈ -Z 轴 | 22.0 |
| J3 | [0, 0.01, -0.9999] ≈ -Z 轴 | 17.0 |
| J4 | [1, 0, 0] = X 轴 | 3.4 |
| J5 | [0, 0.01, -0.9999] ≈ -Z 轴 | 3.4 |
| J6 | [0, 0.9999, 0.01] ≈ Y 轴 | 0.0 (纯姿态) |

### 当前 IK 状态

- 3/5 简单姿态收敛（误差 < 1mm）
- 2/5 边界姿态因 Jacobian 噪声失败
- J6 是纯姿态关节，数值差分测不出，Jacobian 列全是噪声
- 解析 Jacobian 无此问题（ω_6 = Y 轴始终有效）

### PoE 可行性

- 串行运动链天然适合 PoE 公式
- Screw 参数（ω_i, q_i）在挂载时从场景零位提取一次
- FK 和 Jacobian 全部解析计算，无浮点噪声
- 可在 Node.js vitest 中运行
