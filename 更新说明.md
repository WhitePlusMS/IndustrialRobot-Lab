# 更新说明

## 2026-06-18 提升 RX/RY/RZ 姿态控制重复精度

- 问题：连续点击 RX/RY/RZ 方向键时，每次 10° 步进的实际到达角度不一致，重复精度差
- 原因：
  - `solveIK` 收敛 tolerance 较宽松（位置 1.5mm、姿态 0.02rad ≈ 1.15°），每次求解结果在 tolerance 内就停止，实际姿态与目标有偏差
  - `orientationScale` 仅 120，姿态权重偏低，LM 更关注位置收敛
  - 数值 Jacobian 步长 0.4° 较大，导数噪声导致收敛振荡
  - 求解后直接走关节空间缓动，是开环执行，不再修正
- 修复：
  - `src/lib/glb-robot-model.ts`：数值 Jacobian 步长从 0.4° 降到 0.2°，降低导数噪声
  - `src/hooks/useRobot.ts`：姿态移动与 `goToPosition` 的 `solveIK` 参数收紧为：
    - `posTolerance: 0.3mm`（姿态）/ `0.5mm`（定位）
    - `oriTolerance: 0.005rad`（约 0.3°）
    - `orientationScale: 300`，让姿态与位置权重更平衡
    - `damping: 0.15`、`maxStepRad: 0.05`、`maxIterations: 100`
- 验证：`npm run check`、`npm run test`、`npm run build` 均通过

## 2026-06-18 修复 RX/RY/RZ 持续点击可到奇形怪状状态且不报 unreachable 的 Bug

- 问题：点击 RX/RY/RZ 方向键（步进 10°）持续运动时，机械臂会逐渐扭曲到奇怪姿态，且从不报 `unreachable`
- 原因：新的 `solveIK` / `solvePositionOnlyIK` 在迭代中没有把关节行程作为硬约束，会收敛到超出关节限位的解；`useRobot.ts` 求解后再 `clampJointsToRanges` 截断，导致实际运动位姿与目标位姿不符，但系统仍认为“有解”。连续点击时误差不断累积，最终手腕拧成异常姿态
- 修复：
  - `src/lib/ik-solver.ts`：`solveIK` / `solvePositionOnlyIK` 新增可选 `jointRanges` 参数，迭代生成的 `candidate` 立即 clamp 到关节行程；初始 seed 也先做 clamp
  - `src/lib/motion-planner.ts`：`solveCartesianStep` / `solveCartesianPositionStep` 透传 `jointRanges` 到求解器
  - `src/hooks/useMotion.ts`：从 `config.dhParams` 提取 `jointRanges` 传给笛卡尔步进求解
  - `src/hooks/useRobot.ts`：姿态移动与 `goToPosition` 调用 `solveIK` 时传入 `ranges`，让 IK 在关节可行空间内搜索
- 验证：`npm run check`、`npm run test`、`npm run build` 均通过

## 2026-06-18 实现 ref + useFrame 解耦 slider 与 React state

- 问题：滑块拖动时每次 `onChange` 都同步 `setJoint`，虽然目标密集，但 React 每帧重绘整个 UI（关节卡片、数据面板、3D 场景等），主线程被占满，快速拖动时机械臂仍感瞬移/卡顿
- 解决：slider 拖动过程不再触发 React state 更新，而是直接写入 `sliderTargetRef.current`
  - `App.tsx` 创建共享的 `sliderTargetRef`，同时传给 `useRobotKinematics`、`RobotScene` 和 `OperationPanel`
  - `JointAngleCard` 拖动时只更新 `sliderTargetRef.current` 和本地显示；松手后才调用 `onSetJoint` 同步 React state
  - `GLBRobotArm` 在 `useFrame` 中直接读取 `sliderTargetRef.current` 作为插值目标，完全绕过 React 渲染链路
- 效果：拖动滑块时 3D 机械臂在渲染循环内即时跟手，React 仅在不拖动时低频同步，解决快速拖动瞬移问题
- 涉及文件：`App.tsx`、`useRobotKinematics.ts`、`RobotScene.tsx`、`GLBRobotArm.tsx`、`JointAngleCard.tsx`
- `npm run check` 通过，无 TypeScript 错误
