右侧面板 redesign 方案 - 自由练习模式

基于“操作逻辑应优先、减少教学步骤卡片干扰”的原则，提供 5 个 HTML 原型供对比。

现状问题分析

  • 自由模式下仍强行展示教学步骤卡片（关节认识、坐标系说明、单关节滑块等），占用大量空间。
  • 原来的核心操作卡片（JointAngleCard、PoseControlCard、WaypointPanel）被压到最底部，操作路径变长。
  • 相机/抓取模块同样把步骤说明放在操作控件之上， veteran 用户需要反复滚动。
  • 自由练习模式的核心诉求是：快速调出对应功能、直接操作、状态可见。

设计原则

  • 保留你原来 ControlPanel 的 tab 分组逻辑（机器人 / 相机 / 动作编排）。
  • 去掉教学文案和步骤任务卡片，所有控件默认展开，减少点击层级。
  • 高频操作（关节、位姿、吸盘、运行）放在首屏；低频/高级参数可折叠。
  • 状态实时可见：当前坐标系、吸盘状态、运行状态等用 status bar 常驻。
方案 1：经典 Tabs 推荐

直接复刻你原来 ControlPanel 的“机器人控制 / 相机控制 / 动作编排”三段 Tab。每个 Tab 内用可折叠卡片承载控件，结构清晰、学习成本低。

方案 2：全展开 Accordion 紧凑

不切换 Tab，所有功能按模块折叠排列在一个面板内。适合大屏幕或希望一眼看到全部功能的用户；小屏需要滚动。

方案 3：模块 Tabs + 快捷工具栏 现代

保留 Tab 切换，但在每个 Tab 顶部增加一行常驻快捷按钮（如回零、回原点、重置相机、运行序列）。高频操作一键直达，下方再展开细节。

方案 4：扁平 Dock 工具面板 极简

弱化“卡片”概念，把功能区做成一组组按钮块。每个块只有标题和按钮网格，信息密度最高，适合熟练用户快速操作。

我的建议

  • 如果你希望改动最小、保留原操作习惯：选方案 5 或方案 1。它们都保留了你原来的 Tab + 可折叠卡片结构，只是去掉教学步骤卡片、把核心控件前置。
  • 如果你希望首屏操作效率最高：选方案 3。顶部快捷工具栏让回零、重置相机、运行序列等高频操作一键直达。
  • 如果你希望一屏看到所有功能：选方案 2。但面板会较长，需要滚动。
  • 如果你追求极简、按钮化：选方案 4。但对新手不够友好。

请看完这几个 HTML 原型后告诉我倾向哪一版（或哪几版元素组合），我再把它落地到 React 组件里。
